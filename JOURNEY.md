# Schrodinger TTS Detector — Journey

TTS/딥페이크 음성 탐지기의 개발 과정 기록입니다.

---

## 시작

Schrodinger는 원래 마우스 궤적 기반 에임핵 탐지 시스템이었습니다.

핵심은 단순합니다: **물리적 실체가 있는 시스템과 없는 시스템은 다릅니다.**

손은 질량이 있고, 소프트웨어는 질량이 없습니다. 성대도 마찬가지입니다.

## 실패의 연속

처음에는 75% 정도였습니다. 쓸 수 없는 수준이죠.

10개 이상의 접근을 시도했고, 대부분 실패했습니다.
인간과 TTS의 분포가 겹쳐서 구분이 안 됐습니다.

AI 생성 음악도 시도해봤지만, 음악에는 이 접근이 통하지 않았습니다.

## 돌파구

물리학에서 답을 찾았습니다.

성대는 물리 법칙이 지배하는 시스템이고, TTS는 아닙니다.
문제를 "풀기"가 아니라 "검증"의 관점으로 바꾸니까, 방법이 보이기 시작했습니다.

여러 독립적인 물리 제약을 조합해서 하나의 점수로 만들었습니다.

97%까지 올라갔습니다.

## 벽

테스트 샘플을 늘리니 85%로 떨어졌습니다. 일부 화자가 계속 오분류됐습니다.

여기서 포기할 수도 있었는데, 한 가지 의문이 생겼습니다.

**"이거 결국 데이터 양 문제 아닌가?"**

## 버그 한 줄

코드를 다시 봤더니, 핵심 계산에서 입력 데이터의 대부분을 버리고 있었습니다.
480초짜리 오디오를 넣어도 극히 일부만 사용하는 구조였습니다.

한 줄 고쳤습니다.

## 결과

```
Human  anomaly_score:  min=0.2855  max=1.5129  (n=40)
TTS    anomaly_score:  min=0.0585  max=0.2636  (n=21)
Accuracy: 61/61 (100.0%)
```

겹치는 구간이 없습니다.

## 배운 것

1. 10번 실패해도 1번 맞으면 됩니다.
2. 막혔을 때 "왜?"를 한 번 더 물어봐야 합니다.
3. 버그 한 줄이 전체 성능을 결정할 수 있습니다.
4. 포기하지 않으면 됩니다.
